# ADHD Planner — спецификация для реконструкции

**Статус:** рабочий документ, пополняется по шагам.
**Источник:** пояснения автора + скриншоты из чата.

---

## 0. Позиционирование продукта

> **Не просто планер, не менеджер задач. Это СДВГ-брейндамп и личный операционный директор, который помогает не развалиться в жизни.**

Отсюда продуктовый тон: **эмпатия, юмор, живые маскоты, игра в тамагочи** — и при этом **системность** под капотом.

- **Целевой форм-фактор:** мобильное приложение (mobile-first).
- Сейчас есть только десктопный web, мобильной вёрстки нет.

---

## 1. Маскоты = AI-ассистенты

Два кота с разной ролью и голосом. Это и **бренд-маскоты**, и **два AI-канала**.

### 1.1 Ангел (белый кот с нимбом и микрофоном)

- **Позитивный / поддерживающий** голос.
- Ведущий сценария **брейн-дампа**: пользователь наговаривает/пишет поток — **ангел сам режет на задачи**.
- UI: модалка **Angel Lab (beta)** — тёмная, жёлтые обводки, кнопка «Говорить», поле ввода, действия **Назад / Сохранить дамп**, блоки **Дамп** и **Предложенные задачи**.

### 1.2 Чертик (чёрный кот с рожками, трезубец)

- **Провокатор / голос разума**. «А что это за задача? Что за бред?» — может усомниться в «дебильной» задаче.
- **Альтернативный AI-канал** для работы (если позитив ангела не заходит).
- Имеет **свой чат** (чёрный попап внизу: *«Привет! Покажу твои задачи и помогу разобраться с чего начать. Что беспокоит больше всего?»*).
- Может **удалять задачи из Чистилища в Кладбище** — иногда автоматически, иногда по просьбе.
- Когда ⚡ (очки) уходят в минус — приходит: «всё, финита».

### 1.3 Размещение

- На доске: ангел — низ слева, чертик — низ справа (плавающие аватары + реплики).
- В hero-экранах/пустых состояниях — могут подниматься наверх.

## 2. Модель задач — состояния и жизненный цикл

### 2.1 Верхние табы = КАТЕГОРИИ состояния

| Таб | Что внутри |
|---|---|
| **В процессе** (🔥) | активные, делится на 3 подстатуса (см. 2.2) |
| **Рай** (☁) | завершённые: голубые карточки с `+10 POINTS` и кнопкой **Вернуть в активные** |
| **Кладбище** (🪦) | «умершие» задачи — **могилки с RIP** (см. 2.4) |
| **Прогресс** (📊) | статистика + история + снапшоты (см. 2.5) |

### 2.2 Таб «В процессе» — 3 колонки (подстатусы)

| Колонка | Смысл | Маркеры |
|---|---|---|
| **В фокусе** | юзер помнит и работает | «горячие» задачи |
| **На фоне** | подзабытые, остывают | **строка «горячести»** (деградирует со временем) |
| **Чистилище** | кандидаты на смерть | при неактивности **высыхают** → Кладбище |

### 2.3 Жизненный цикл

```
  Brain dump (Angel Lab)
          │
          ▼
   [ В процессе ]
      ├─ В фокусе  ⇄  На фоне  ⇄  Чистилище
      │                              │
      │  ┌───────── успех ───────────┘
      │  ▼                           │
      │ [ Рай ]  (+10 POINTS, можно вернуть в активные)
      │                              │
      ▼                       высыхание / чертик
   возврат                           ▼
                             [ Кладбище ]
                             (RIP, воскрешение возможно)
```

### 2.4 Кладбище — подробно *(задумано, в коде ещё нет)*

- Карточки-**могилки**: плита, слово **RIP**, **имя задачи**, **дата смерти**.
- **Поэтическая причина смерти** — случайная строка из набора:
  - *«Была отправлена в забвение злыми силами.»*
  - *«Не вынесла тяжёлой ноши бренной жизни.»*
  - *(добавить ещё 6–8 вариантов в стиле эпитафий — для разнообразия)*
- **Воскрешение возможно** — кнопка типа *«Воскресить»* возвращает задачу в активные (в `В фокусе` или `На фоне`).
- Причина попадания: `высохла` (автомат) или `убита чертиком` (ручное действие AI).

### 2.5 Прогресс — статистика

Четыре плитки:

- **⏱ Вложено времени**
- **☁ Завершено задач**
- **⚡ Очков набрано** (см. §4)
- **🔥 Дни подряд**

Плюс: **История задач по дням** (раскрываемый список), **Снапшоты (резервные копии)** — Создать / Загрузить.

### 2.6 Метрики задачи

- **Пульс задачи** — % выполнения. Отсюда кнопка вида **«ДОБАВИТЬ (ПУЛЬС 35)»**.
- **Срочность**, **Страшность** — параметры приоритизации.
- Цвет прогресс-баров по колонкам: зелёный / синий / красный.

## 3. Брейн-дамп и ввод

- **«Выгрузить из головы»** (в шапке) и **оранжевая вертикальная вкладка AI** (справа у края) — **обе ведут в Angel Lab**.
- **Быстрый ввод снизу:** «Какую глобальную задачу берём на прицел?» + **ДОБАВИТЬ (ПУЛЬС N)**.

---

## 4. ⚡ Молния — система баллов

- Счётчик в шапке (напр. `768`).
- **Может уходить в минус** — если задачи умирают.
- Уход в минус триггерит явление чертика.

---

## 5. Пинки — напоминания

- Каналы: **Telegram**, **e-mail**, **push**, **браузерные нотификации**.
- UI: **Включить пинки**, **Тестовый пинок** (в колонке действий миссии).

## 6. Режимы

### 6.1 Тема — «Луна» 🌙

Переключатель светлой / тёмной темы.

### 6.2 Концентрация — «Туман»

- Кнопка тумана рядом с луной.
- **Открывает режим концентрации.**
- **Второй вход:** клик по блоку **TODAY MISSION**.
- **Панические режимы объединяются.** Сейчас раздельно: *Panic mode* и *Focus*. В новой версии → **единый режим** со сценарием Panic Mode (см. §7) как «жёсткий» подрежим.

### 6.3 Интеграция календаря — иконка «страница N» (напр. 17)

- Подключение **Google Calendar** (в перспективе — других календарей).
- Число = текущий день? (пока допустим **день месяца**, уточнить).

## 7. Panic Mode (ключевой UX)

**Задача:** снять аналитический паралич и запустить движение, не требуя от мозга решения «что делать».

**Состав экрана (с реального скрина):**

- Заголовок **`PANIC MODE`** (моно/пиксель).
- Имя задачи крупно, пиксельным шрифтом: *Выставить свитер Stone Island на продажу*.
- Лид: *«У задачи пока нет подпунктов. Значит цель сейчас — не сделать её, а всего лишь запустить движение.»*
- **3 пронумерованные карточки-шага** (с оранжевыми цифрами в квадрате):
  1. Откройте всё, что связано с задачей «[имя]».
  2. Сделайте **один уродливый первый шаг** на 2 минуты.
  3. Если станет легче — сразу добавьте первый подпункт прямо из panic mode.
- Плашка-девиз: **`НУЖЕН ТОЛЬКО МИКРОСДВИГ`** + таймер **`2:00`**.
- Поле «*Если мозг кипит, впиши сюда самый крошечный следующий шаг*» + кнопка **+ Добавить шаг**.
- 4 кнопки:
  - **Запустить 2 минуты** (primary, розовая).
  - **✅ Я сдвинулась** (success).
  - **🎯 Показать задачу**.
  - **Позже**.

**Голос копирайта:** прямой, по-дружески, на «ты»/«вы» вперемешку («впиши сюда», «если станет легче»). Сохранять этот голос во всех состояниях.

---

## 8. Блок TODAY MISSION

- Пиксельный заголовок задачи.
- Статус срока («срок прошёл…»).
- Цветные теги: `сроки`, `жёсткий дедлайн`, `жизненно важно`, `срочно`, мета-чипы.
- Колонка действий справа: **Спасти сейчас**, **Показать цель**, **Раскипеть** *(OCR — сверить)*, **Включить пинки**, **Тестовый пинок**.
- **Клик по блоку → Focus/Panic**.

## 13. Стиль финальной версии — решено

**Выбран гибрид (option C):**

- **Душа — твоя:** пиксель-дисплей для чисел/таймеров, chunky pill-badges, лавандово-розовый паспорт цвета, маскоты-коты, «тамагочи»-тон.
- **Скелет — из Design Guide 2026:** index/catalog сетка, строгая типографическая иерархия (один editorial serif для «имени задачи», mono для системных лейблов вида `FIG.` / `TODAY MISSION`), sensory-темнота в Focus/Panic, дисциплина воздуха.
- **Scrapbook-вставки** локально и дозированно: тейп на ярлыке «новое», рукописный штрих у реплик маскотов, могилки с аналоговой фактурой в Кладбище.

Это даёт: **не теряем тамагочи-ДНК**, но получаем читаемость, hierarchy и «взрослый» слой, который не стыдно нести в App Store.
